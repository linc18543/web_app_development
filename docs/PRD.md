# 個人記帳簿 - 產品需求文件 (PRD)

## 1. 專案概述
- **專案名稱**：個人記帳簿
- **背景與動機**：現代人生活步調緊湊，容易在不知不覺中產生許多零碎開銷。本專案旨在提供一個直觀、便捷的介面，協助使用者追蹤個人的收支狀況、管理預算，並透過視覺化圖表分析財務健康度。
- **目標用戶**：學生、上班族、有理財規劃需求的人士，或是需要幫家庭或長輩代為記帳的使用者。
- **核心價值主張**：讓記帳變得簡單無負擔，快速掌握現金流向，協助達成個人與家庭理財目標。

## 2. 功能需求

以下為系統的主要功能及其對應的使用者故事：

### 2.1 收入與支出紀錄
- **使用者故事**：作為一名使用者，我希望能夠快速新增、編輯和刪除收入與支出的紀錄（包含金額、日期、分類、備註等），以便準確追蹤我的金流。

### 2.2 分類統計分析
- **使用者故事**：作為一名使用者，我希望系統能將我的收支依照不同的類別進行加總與統計，以便我快速了解各種項目的花費比例與習慣。

### 2.3 預算設定
- **使用者故事**：作為一名使用者，我希望可以設定每個月或不同類別的支出預算上限，以便在我快要超支時能夠有所警覺並控制開銷。

### 2.4 報表與圖表功能
- **使用者故事**：作為一名使用者，我希望系統能將收支資料以視覺化的圖表（如圓餅圖、長條圖）與報表呈現，以便我進行更直覺的長期財務分析。

### 2.5 資料備份與同步
- **使用者故事**：作為一名使用者，我希望記帳資料能夠被安全地備份且支援匯出或同步，以便在更換設備或發生意外時不會遺失個人的財務紀錄。

## 3. 非功能需求

- **技術限制**：
  - 前端：使用 HTML / CSS / JavaScript 搭配 **Jinja2** 模板。
  - 後端：使用 Python 的 **Flask** 框架。
  - 資料庫：使用 **SQLite** 作為關聯式資料庫。
- **效能考量**：
  - 確保頁面載入速度快，表單送出與圖表渲染反應迅速，以提供良好的使用者體驗。
- **安全考量**：
  - 若系統開放多人使用，需具備基本的使用者驗證（Authentication）機制。
  - 防範常見的 Web 攻擊（如 SQL Injection, XSS 等），確保使用者財務資料隱私。

## 4. MVP 範圍 (Minimum Viable Product)

- **Must Have (必須有)**
  - 基礎的使用者帳號系統（註冊與登入）。
  - 新增、修改、刪除收入與支出的功能，並能指定分類（如餐飲、交通、薪水等）。
  - 顯示當月總收入、總支出與結餘的儀表板。

- **Should Have (應該有)**
  - 預算設定功能（設定月度總預算並顯示進度條）。
  - 基礎圖表呈現（圓餅圖分析主要支出類別）。

- **Nice to Have (加分項)**
  - 多圖表分析與報表匯出（CSV等）。
  - 各種進階篩選條件（依日期區間、關鍵字搜尋）。
  - 資料備份與雲端同步機制。
